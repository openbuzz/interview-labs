interview-labs workspace

A sandboxed VS Code + terminal environment for this session.

Get started

Open VS Code Open terminal

VS Code is the primary editor — files, extensions, integrated terminal, debugger. The standalone terminal is a fast fallback when the in-editor one feels slow. Both open in ~/workspace.

How this works

Tasks mirror real work: a real editor, a real filesystem, a real shell — not a toy sandbox or a quiz. Find the brief in ~/workspace, then build, run, and test exactly as you would on the job.

Available tools

  • Editor — VS Code (code-server)
  • OS — Ubuntu 26.04
  • Shell — passwordless sudo, bash completion enabled
  • DevTools — git, vim, curl

Need more? Customize the environment yourself (e.g. sudo apt install …) — no need to go overboard.

Good to know

  • This session is destroyed after the interview — nothing here persists.
  • Don't bring personal or sensitive data into the session.
  • Clipboard in browser VS Code needs a secure context: over HTTPS (or localhost) copy/paste works normally; over plain HTTP the browser blocks the clipboard API. The editor's Ctrl/Cmd+C/V still work, but the terminal needs Ctrl+Shift+C / Ctrl+Shift+V — allow the browser's clipboard prompt on first paste.
